## 🐛 자주 발생하는 오류와 해결 방법

### 오류 1: "구글 시트 설정이 필요합니다"

**증상:**
- study-card.html을 열면 빨간색 오류 메시지가 표시됨
- "구글 시트 설정이 필요합니다"라는 메시지

**원인:**
- admin.html에서 초기 설정을 하지 않았음
- 또는 다른 브라우저/시크릿 모드에서 열어서 설정이 없음

**해결 방법:**
1. `admin.html` 파일을 브라우저로 엽니다
2. "초기 설정" 탭에서:
   - 구글 시트 ID 입력
   - API Key 입력
   - 시트 이름 확인 (기본값: Sheet1)
3. "설정 저장" 버튼 클릭
4. 다시 study-card.html 열기

**중요:** admin.html과 study-card.html을 **같은 브라우저**에서 열어야 합니다!

---

### 오류 2: "데이터를 불러올 수 없습니다"

**증상:**
- 로딩 후 오류 메시지 표시
- 네트워크 오류 또는 401/403 에러

**원인 및 해결:**

#### 2-1. 구글 시트 공유 설정 문제
✅ **해결:**
1. 구글 스프레드시트 열기
2. 오른쪽 상단 "공유" 버튼 클릭
3. "링크가 있는 모든 사용자" 선택
4. 권한: "뷰어" (읽기 전용)
5. "완료" 클릭

#### 2-2. API 키 오류
✅ **해결:**
1. API 키를 다시 복사해서 입력했는지 확인
2. 앞뒤 공백이 없는지 확인
3. Google Cloud Console에서 API 키 상태 확인:
   - https://console.cloud.google.com
   - API 및 서비스 → 사용자 인증 정보
   - 해당 API 키 클릭
   - "활성" 상태인지 확인

#### 2-3. Google Sheets API 비활성화
✅ **해결:**
1. Google Cloud Console 접속
2. API 및 서비스 → 라이브러리
3. "Google Sheets API" 검색
4. "사용 설정됨" 확인 (아니면 "사용" 클릭)

#### 2-4. 시트 ID가 잘못됨
✅ **해결:**
시트 URL에서 정확한 ID 복사:
```
https://docs.google.com/spreadsheets/d/1a2b3c4d5e6f7g8h9i0j/edit
                                      ↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑
                                      이 부분만 복사
```

### 오류 3: "해당 학생의 학습 기록을 찾을 수 없습니다"

**증상:**
- 구글 시트 연동은 되는데 특정 학생 데이터를 못 찾음

**원인 및 해결:**

#### 3-1. 학생 이름 불일치
구글 시트의 학생 이름과 링크의 학생 이름이 정확히 일치해야 합니다.

예시:
- 시트: "홍길동"
- 링크: "홍 길동" ❌ (띄어쓰기)
- 링크: "홍길동" ✅

✅ **해결:**
- 띄어쓰기, 대소문자 정확히 확인
- 특수문자 확인 (예: "․" vs ".")

#### 3-2. 날짜 형식 불일치
날짜는 반드시 **YYYY-MM-DD** 형식이어야 합니다.

예시:
- 시트: "2024-11-06" ✅
- 시트: "2024/11/06" ❌
- 시트: "11-06" ❌
- 시트: "24-11-06" ❌

✅ **해결:**
구글 시트의 날짜 열을 "텍스트" 형식으로 변경:
1. B열 전체 선택
2. 서식 → 숫자 → 일반 텍스트
3. 날짜를 YYYY-MM-DD 형식으로 입력

#### 3-3. 빈 행이 있음
데이터 중간에 빈 행이 있으면 그 이후 데이터를 읽지 못할 수 있습니다.

✅ **해결:**
- 빈 행 삭제
- 데이터를 첫 행부터 연속으로 입력

### 오류 4: 로고 이미지가 안 보임

**증상:**
- 카드는 잘 나오는데 로고 이미지가 표시되지 않음

**원인 및 해결:**

#### 4-1. 파일 위치 문제
✅ **해결:**
logo.jpg 파일이 HTML 파일과 **같은 폴더**에 있어야 합니다:
```
📁 my-folder/
  ├── study-card.html
  ├── admin.html
  ├── demo.html
  └── logo.jpg  ← 같은 위치!
```

#### 4-2. 파일명 문제
✅ **해결:**
- 정확히 "logo.jpg" (소문자)
- "Logo.jpg" ❌
- "logo.JPG" ❌
- "logo.png" ❌ (png면 HTML 수정 필요)

#### 4-3. GitHub Pages에서 안 보임
✅ **해결:**
1. GitHub 저장소에 logo.jpg가 업로드 되었는지 확인
2. 파일명 대소문자 정확히 확인
3. 5-10분 정도 기다렸다가 다시 확인

---

### 오류 5: 링크 복사가 안 됨

**증상:**
- "링크 복사" 버튼을 클릭해도 아무 반응 없음

**원인:**
- HTTP(보안되지 않은 연결)에서는 clipboard API가 작동하지 않을 수 있음

**해결:**
1. **Chrome/Edge**: 보안 경고 무시하고 계속 진행
2. **수동 복사**: 링크 텍스트를 마우스로 드래그해서 Ctrl+C
3. **HTTPS 사용**: GitHub Pages 등 HTTPS 환경에서 사용

---

### 오류 6: 모바일에서 레이아웃이 깨짐

**증상:**
- 스마트폰에서 보면 글자가 너무 작거나 화면 밖으로 나감

**원인:**
- 브라우저 캐시 문제

**해결:**
1. 모바일 브라우저 새로고침 (Ctrl+R)
2. 캐시 삭제 후 다시 접속
3. 다른 브라우저로 테스트 (Chrome, Safari)

## 📱 로컬 vs 온라인 테스트

### 로컬 테스트 (컴퓨터에서만)
**장점:**
- 설정 간단
- 빠른 테스트

**단점:**
- 다른 사람에게 링크 전송 불가
- 스마트폰에서 확인 불가

**사용법:**
1. HTML 파일들을 한 폴더에 저장
2. admin.html을 브라우저로 열기
3. 설정 후 테스트

### 온라인 배포 (GitHub Pages)
**장점:**
- 누구나 링크로 접근 가능
- 스마트폰에서도 확인 가능
- 카카오톡 전송 가능

**단점:**
- 초기 설정 필요
- 파일 수정 시 재업로드 필요

**권장:** 실제 사용은 GitHub Pages로!

### 3단계: API 확인
- [ ] Google Cloud Console 접속됨
- [ ] 프로젝트 생성됨
- [ ] Google Sheets API 활성화됨
- [ ] API 키 생성됨
- [ ] API 키 복사해둠

### 4단계: admin.html 설정
- [ ] admin.html을 브라우저로 열림
- [ ] "초기 설정" 탭에서:
  - [ ] 구글 시트 ID 입력함

## 🆘 그래도 안 되면?

### 가장 간단한 해결책
1. **모든 파일 삭제**
2. **처음부터 다시 시작**
3. **README.md 단계별로 정확히 따라하기**

### 추가 도움
- 브라우저 콘솔(F12)의 오류 메시지 확인
- 구글 시트 공유 링크가 실제로 작동하는지 확인 (링크 복사해서 시크릿 모드로 열기)
- API 키의 할당량 확인 (Google Cloud Console)

## 💡 Pro Tips

1. **테스트용 시트 만들기**: 실제 데이터 입력 전에 테스트용 시트로 먼저 연습
2. **학생 이름 표준화**: "성+이름" 형식으로 통일 (예: 홍길동, 김영희)
3. **날짜 자동 입력**: 구글 시트에서 `=TODAY()` 함수 활용
4. **템플릿 복사**: 첫 줄 데이터를 복사해서 나머지 채우기
5. **백업**: 구글 시트는 자동 저장되지만 중요한 설정은 메모장에 기록
